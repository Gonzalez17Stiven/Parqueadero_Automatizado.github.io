EJERCICIO VALIDADOR DATOS FORMULARIO

EJERCICIO VALIDACIÓN DE DATOS 3


Este ejercicio puede ser utilizado como ejemplo para implementar controles de verificación de DATOS ingresados en un formulario de HTML5. Aquí puede observar algunos procedimiento simples utilizando "Expresiones Regulares" que suelen aplicarse para validar la consistencia de los datos capturados y para asegurar que los datos estén correctos antes de su almacenamiento o procesamiento.

Sirve como tutorial y referencia para conocer el lenguaje de programación Java Script y su utilidad en el desarrollo de páginas WEB.

En este formulario se registran cinco datos con características y condiciones de validación diferentes:

  • Nombre y Apellido: Dato alfanumérico, incluye espacios en blanco, letras con acentos y otros caracteres especiales como la "ñ". Debe empezar por una letra y no puede contener algunos símbolos como el $, @, %, entre otros.
  • Edad: Dato numérico (positivo) mayor que 10 y menor que 100.
  • Fecha de Nacimiento: Dato tipo Fecha en formato dd/mm/aaaa debe ser una fecha válida según calendario de cada mes.
  • Género: Dato tipo Caracter, mediante selección entre varias opciones (F o M), controla que se haya elegido alguna las opciones disponibles.
  • Correo Electrónico: Texto con una dirección de correo electronico en formato usuario.usuario@dominio.extension
  • Número Telefónico: cifra numérica entera positiva, debe tener como mínimo 7 dígitos de longitud
  • Tipo de Número Telefónico: listado de selección mediante cuadro combinado (lista desplegable)
  • Mensaje de texto: Mensaje de texto largo que puede ocupar varios renglones, de máximo 1000 caracteres de longitud (textarea)
  • Aceptación de términos y condiciones: Caja de chequeo, mediante la cual se marca o se desmarca la aceptación de un dato, en este caso el tratamiento de datos para aceptar envío del formulario
